# NPU 硬件架构规格

## 1. 概述

AI Core AI 处理器的计算核，负责执行矩阵、矢量计算密集的任务。基于 PyPTO 开发的算子运行在 AI Core 上。

## 3. 计算单元

### 3.1 计算单元概览

| 单元 | 功能 | 数据来源 |
|------|------|----------|
| **Cube** | 矩阵运算 (如 16×16 矩阵乘法) | L1 Buffer (左矩阵/累加), L0B Buffer (右矩阵) |
| **Vector** | 向量运算 (SIMD) | Unified Buffer |
| **Scalar** | 标量运算、流程控制、指令发射 | GM (通过 ICache/DCache) |

### 3.2 Cube 计算单元

```
┌────────────────────────────────────────────────┐
│              Cube 计算单元数据流                │
├────────────────────────────────────────────────┤
│                                                │
│   L1 Buffer           L0B Buffer               │
│   (左矩阵/累加)        (右矩阵)                 │
│       │                   │                    │
│       └───────┬───────────┘                    │
│               ▼                                │
│         ┌───────────┐                          │
│         │   Cube    │  ◀── 矩阵乘法            │
│         │  计算单元  │     float16: 16×16      │
│         └─────┬─────┘                          │
│               │                                │
│               ▼                                │
│         L1 Buffer                              │
│         (结果+中间结果)                         │
│                                                │
└────────────────────────────────────────────────┘
```

### 3.3 Vector 计算单元

- 执行 SIMD (单指令多数据) 向量指令
- 支持多次迭代执行
- 支持带间隔向量运算
- **对齐要求：32 字节**

```
┌────────────────────────────────────────────────┐
│             Vector 计算单元数据流               │
├────────────────────────────────────────────────┤
│                                                │
│         Unified Buffer                         │
│         (源数据 + 目标数据)                     │
│               │                                │
│               ▼                                │
│         ┌───────────┐                          │
│         │  Vector   │  ◀── 向量加/乘等         │
│         │  计算单元  │     首地址: 32B对齐      │
│         └─────┬─────┘                          │
│               │                                │
│               ▼                                │
│         Unified Buffer                         │
│                                                │
└────────────────────────────────────────────────┘
```

### 3.4 Scalar 计算单元

- 类似小型 CPU
- 功能：循环控制、分支判断、地址/参数计算、指令发射
- 通过事件同步模块控制流水线

```
┌────────────────────────────────────────────────┐
│              Scalar 对指令和数据的访问          │
├────────────────────────────────────────────────┤
│                                                │
│   ┌───────────┐      ┌───────────┐            │
│   │   ICache  │      │   DCache  │            │
│   │ (代码缓存) │      │ (数据缓存) │            │
│   │  16K/32K  │      │   16K     │            │
│   └─────┬─────┘      └─────┬─────┘            │
│         │                  │                  │
│         ▼                  ▼                  │
│   ┌─────────────────────────────┐             │
│   │          Scalar             │             │
│   │         (ALU)               │             │
│   └─────────────────────────────┘             │
│                                                │
│   ICache: 以 2K 为单位加载                      │
│   DCache: 以 Cache Line (64B) 为单位加载       │
│                                                │
└────────────────────────────────────────────────┘
```

### 4.2 存储单元详细规格

| 存储单元 | 描述 | 对齐要求 (300x) | 推荐格式 |
|----------|------|-----------------|----------|
| **L1 Buffer** | 通用内部存储，数据中转区，Cube 左矩阵/累加输入输出 | 32 Byte | FRACTAL_NZ |
| **L0B Buffer** | Cube 右矩阵输入 | 512 Byte | FRACTAL_ZN |
| **Unified Buffer** | 向量/标量计算输入输出 | 32 Byte | 无要求 |
| **BiasTable Buffer** | 存放矩阵计算 Bias | 64 Byte | - |
| **Fixpipe Buffer** | 存放量化和 ReLU 参数 | 64 Byte | - |

## 5. 搬运单元 (MTE)

### 5.1 搬运单元功能

| 搬运单元 | 数据通路 | 说明 |
|----------|----------|------|
| **MTE1** | L1 → L0B | Cube 数据准备 |
| **MTE2** | GM → {L1, UB} | 外部数据加载，基于分形/Cache Line 对齐更优 |
| **MTE3** | UB → {GM, L1}；L1 → GM | 结果写回 |
| **FixPipe** | L1 → L1, L1 → BT Buffer, L1 → FP Buffer, L1 → UB | 支持随路数据格式/类型转换 |

### 5.2 搬运对齐要求

```
┌────────────────────────────────────────────────────────────┐
│                     搬运对齐规则                            │
├────────────────────────────────────────────────────────────┤
│                                                            │
│  搬运到 Unified Buffer:                                    │
│    └─▶ 按 DataBlock 对齐                                   │
│                                                            │
│  搬运到其他存储单元 (L0B 等):                             │
│    └─▶ 按分形 (Fractal) 大小对齐                           │
│                                                            │
│  示例: L1 → L0B                                            │
│    └─▶ 数据格式: NZ → ZN                                   │
│    └─▶ 搬运大小: 按分形大小对齐                             │
│    └─▶ 若 L1 剩余空间 < 1 分形 → 硬件异常                   │
│                                                            │
└────────────────────────────────────────────────────────────┘
```

### 5.3 FixPipe 随路处理能力

在讨论 Cube → Vector 级联之前，需要理解 **FixPipe 的随路处理能力**。许多后处理操作可以在 Cube 计算过程中由 FixPipe 同步完成，无需进入独立的 Vector 计算阶段。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    FixPipe 随路处理 vs 独立 Vector 算子                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   FixPipe 可随路处理 (无需独立 Vector 阶段):                                  │
│   ───────────────────────────────────────────                               │
│   ✓ 量化 / 反量化                                                           │
│   ✓ ReLU, LeakyReLU 等简单激活                                              │
│   ✓ Bias 加法                                                               │
│   ✓ 格式转换                                                                │
│                                                                             │
│   这些操作通过配置 Fixpipe Buffer 参数即可，视为 Cube 计算单元的能力           │
│                                                                             │
│                                                                             │
│   需要独立 Vector 算子 (需要 Cube→Vector 级联):                               │
│   ───────────────────────────────────────────                               │
│   ✓ Softmax                                                                 │
│   ✓ LayerNorm, BatchNorm                                                    │
│   ✓ GELU, SiLU 等复杂激活 (视硬件支持情况)                                   │
│   ✓ 一般向量计算 (Add, Mul, Div 等)                                         │
│                                                                             │
│   这些操作需要 Cube 结果通过 FixPipe (L1→UB) 后，进入独立 Vector 阶段          │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**关键区别：**

| 特性 | FixPipe 随路处理 | 独立 Vector 算子 |
|------|-----------------|-----------------|
| 执行位置 | Cube 流水线内部 | 独立 Vector 计算阶段 |
| 数据通路 | L1 内部处理 | L1 → FixPipe → UB → Vector |
| 配置方式 | 配置 Fixpipe Buffer 参数 | 编写独立 Vector 计算代码 |
| 典型操作 | 量化、ReLU、Bias | Softmax、LayerNorm、复杂计算 |

**对级联模式的影响：**

- **量化推理 (INT8 MatMul)**：量化和反量化可通过 FixPipe 随路完成，**不需要** Vector → Cube → Vector 三级级联
- **简单激活后处理**：ReLU 等可通过 FixPipe 随路完成，**不需要** Cube → Vector 级联
- **复杂后处理**：Softmax、LayerNorm 等需要独立 Vector 计算，**需要** Cube → Vector 级联

## 6. 数据流

### 6.1 Vector 计算数据流

```
    方式1: GM 数据通过 MTE2 指令搬到 UB，Vector 计算从 UB 中取数据计算，
           计算后的结果写回 UB，最终通过 MTE3 指令将 UB 中的数据写到 GM。

    方式2: GM 数据通过 MTE2 指令搬到 UB，Vector 计算从 UB 中取数据计算，
           计算后的结果写回 UB，最终通过 MTE3 指令将 UB 中的数据写到 L1。

    方式3: L1 数据通过 FixPipe 指令搬到 UB，Vector 计算从 UB 中取数据计算，
           计算后的结果写回 UB，最终通过 MTE3 指令将 UB 中的数据写到 L1 或 GM。

           当 Cube+Vector 级联时，Cube 的计算结果写到 L1，
           通过 FixPipe 直接搬运到 UB 进行 Vector 计算，无需经过 GM。


    ┌──────────────────────────────────────────────────────────┐
    │                                                          │
    │   GM ──(MTE2)──▶ UB ──(Vector)──▶ UB ──(MTE3)──▶ GM      │
    │                 ▲                       │                │
    │                 │                       ├────▶ L1        │
    │                 │                       │    (方式2)     │
    │                 │                       │                │
    │   L1 ──(FixPipe)┘                       └────▶ L1 或 GM  │
    │        (方式3)                               (方式3)     │
    │            ▲                                             │
    │            │                                             │
    │   Cube ────┘  (Cube+Vector 级联，Cube 结果直接作为输入)   │
    │                                                          │
    └──────────────────────────────────────────────────────────┘
```

### 6.2 Cube 计算数据流

#### 基本计算

```
    GM 通过 MTE2 将左矩阵数据搬运到 L1，
    GM 通过 MTE2 将右矩阵搬运到 L1 再通过 MTE1 将数据搬运到 L0B，
    GM 通过 MTE2 将数据搬运到 L1 再通过 FixPipe 搬运到 Fixpipe Buffer，
    数据准备完成后 Cube 读取 L1（左矩阵）、L0B（右矩阵）、Fixpipe Buffer 的数据进行计算，
    计算结果写到 L1。


    ┌──────────────────────────────────────────────────────────┐
    │                                                          │
    │   左矩阵:                                                 │
    │       GM ──(MTE2)──▶ L1 ──┐                               │
    │                            │                              │
    │   右矩阵:                  │                              │
    │       GM ──(MTE2)──▶ L1 ──(MTE1)──▶ L0B ──┐               │
    │                                           │               │
    │   Fixpipe Buffer:                         ├──▶ Cube ──▶ L1│
    │       GM ──(MTE2)──▶ L1 ──(FixPipe)──▶ FP ─┘               │
    │                                                          │
    │   (FP = Fixpipe Buffer)                                  │
    │                                                          │
    └──────────────────────────────────────────────────────────┘
```

#### 累加计算

```
    当存在中间结果时（分块矩阵乘法场景），中间结果是上一次 Cube 计算的结果，
    保存在 L1 上，本次 Cube 计算需要读取这块数据进行累加。


    ┌──────────────────────────────────────────────────────────┐
    │                                                          │
    │   左矩阵 (新):                                            │
    │       GM ──(MTE2)──▶ L1 ──┐                               │
    │                            │                              │
    │   右矩阵 (新):             │                              │
    │       GM ──(MTE2)──▶ L1 ──(MTE1)──▶ L0B ──┐               │
    │                                           │               │
    │   Fixpipe Buffer:                         │               │
    │       GM ──(MTE2)──▶ L1 ──(FixPipe)──▶ FP ─┤               │
    │                                           │               │
    │   累加数据 (中间结果，已在 L1):            │               │
    │       L1 ──(上次Cube结果)──────────────────┤               │
    │                                           │               │
    │                                           ├──▶ Cube ──▶ L1│
    │                                           │    (累加结果)  │
    │   (FP = Fixpipe Buffer)                  │               │
    │                                                          │
    └──────────────────────────────────────────────────────────┘
```

#### Vector+Cube 级联

```
    当 Vector+Cube 级联时，矩阵数据来自 Vector 计算（保存在 UB），
    通过 MTE3 直接搬运到 L1，无需经过 GM。


    ┌──────────────────────────────────────────────────────────┐
    │                                                          │
    │   左矩阵:                                                 │
    │       UB ──(MTE3)──▶ L1 ──┐                               │
    │                            │                              │
    │   右矩阵:                  │                              │
    │       UB ──(MTE3)──▶ L1 ──(MTE1)──▶ L0B ──┐               │
    │                                           │               │
    │   Fixpipe Buffer:                         ├──▶ Cube ──▶ L1│
    │       GM ──(MTE2)──▶ L1 ──(FixPipe)──▶ FP ─┘               │
    │                                                          │
    │   (FP = Fixpipe Buffer)                                  │
    │   (UB 数据来自 Vector 计算)                               │
    │                                                          │
    └──────────────────────────────────────────────────────────┘
```

### 7.2 同步机制

同一个指令序列中的指令是按照进入指令序列的顺序执行的，不同指令序列之间可以并行执行，通过多个指令序列的并行执行可以提升整体执行效率。对于并行执行过程中可能出现的数据依赖，通过事件同步模块插入同步指令来控制流水线的同步。

#### 7.2.1 同一指令序列内的同步

```
┌─────────────────────────────────────────────────────────────┐
│              同一指令序列内的同步 (PipeBarrier)               │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   同一指令序列中的指令按顺序执行，但后一条指令开始时           │
│   前一条指令不一定执行结束。                                 │
│                                                             │
│   PipeBarrier 本身是一条指令，用于约束序列内部的执行顺序，     │
│   保证前序指令中所有数据读写全部完成后，后序指令才开始执行。   │
│                                                             │
│                                                             │
│   指令序列:                                                  │
│                                                             │
│   Op1 ──▶ Op2 ──▶ [PipeBarrier] ──▶ Op3                    │
│                          │                                  │
│                          ▼                                  │
│              等待 Op1, Op2 全部完成                          │
│              然后才开始执行 Op3                              │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 7.2.2 不同指令序列间的同步

```
┌─────────────────────────────────────────────────────────────┐
│            不同指令序列间的同步 (SetFlag/WaitFlag)            │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   不同指令序列可以并行执行，通过 SetFlag/WaitFlag 实现        │
│   序列间的"锁"机制：                                         │
│                                                             │
│   SetFlag：前序操作全部完成后，将硬件中的对应标志位设置为 1   │
│   WaitFlag：标志位为 1 时清零并继续，为 0 时阻塞等待          │
│                                                             │
│                                                             │
│   序列 A:                          序列 B:                   │
│                                                             │
│   Op1 ──▶ Op2 ──▶ SetFlag ──▶    WaitFlag ──▶ Op3 ──▶ Op4  │
│                      │                │                     │
│                      ▼                ▼                     │
│              前序操作完成,        等待 Flag=1,              │
│              设置 Flag=1         清零 Flag, 继续执行         │
│                                                             │
```

## 9. Bank 冲突优化

### 9.1 Bank 冲突问题

当多个操作同时访问 Unified Buffer 的同一个 bank 或 bank group 时，会发生：
- 读写冲突
- 写写冲突
- 读读冲突

这会导致访问排队，降低性能。

### 9.2 优化方案

通过优化 bank 分配方式提升读写性能，详见"避免 Unified Buffer 的 bank 冲突"章节。

## 11. 关键术语

| 术语 | 定义 |
|------|------|
| **Core** | 拥有独立 Scalar 计算单元的计算核 |
| **AI Core** | 昇腾 AI 处理器的计算核，负责矩阵、矢量计算密集任务 |
| **Cube Core** | 矩阵计算核，由 Scalar + 矩阵计算单元 + 搬运单元组成 |
| **Vector Core** | 矢量计算核，由 Scalar + 矢量计算单元 + 搬运单元组成 |
| **分形 (Fractal)** | 数据排布的基本单位，如 16×16 矩阵块 |

## 12. 编程建议

### 12.1 Scalar 优化

- 尽量减少 if/else 分支判断
- 尽量减少变量运算
- 保证代码段被缓存在 ICache
- 保证数据段被缓存在 DCache
- DCache 数据首地址与 Cache Line (64B) 对齐

### 12.2 数据搬运优化

- 基于 Cache Line 大小搬运 (性能更优)
- 基于分形大小搬运
- 满足对齐要求

### 12.3 同步控制

- 推荐使用编程模型自动同步
- 少数情况需手动插入同步
- SetFlag/WaitFlag 必须成对且参数一致

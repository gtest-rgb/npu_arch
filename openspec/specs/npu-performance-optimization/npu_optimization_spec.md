## 1. 性能优化的硬件基础

### 1.1 指令队列并行性 —— 优化的基石

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    指令队列并行性                                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │                    Scalar (指令分类与调度)                           │   │
│   └─┬───────┬───────┬───────┬───────┬───────┬───────┬───┘               │   │
│     │       │       │       │       │       │       │                    │   │
│     ▼       ▼       ▼       ▼       ▼       ▼       ▼                    │   │
│   ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐              │   │
│   │Scalar│ │Vector│ │Cube │ │MTE1 │ │MTE2 │ │MTE3 │ │FixPipe│              │   │
│   │ 序列 │ │ 序列 │ │ 序列 │ │ 序列 │ │ 序列 │ │ 序列 │ │ 序列 │              │   │
│   └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘              │   │
│      │       │       │       │       │       │       │                    │   │
│      └───────┴───────┴───────┴───────┴───────┴───────┘                    │   │
│                          │                                                  │   │
│                          ▼                                                  │   │
│              不同序列间可并行执行                                            │   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**关键洞察：**

- Scalar/Vector/Cube/MTE1/MTE2/MTE3/FixPipe 各为独立指令序列
- 任意两个不同序列之间都可以并行执行
- 这是 DoubleBuffer 优化的硬件理论基础

## 2. Vector 计算流水线优化

### 2.1 问题：串行执行导致 Vector 空闲

```
数据流: GM ──(MTE2)──▶ UB ──(Vector)──▶ UB ──(MTE3)──▶ GM

时间轴 ────────────────────────────────────────────────────────────────────────▶

        │◀───── Block0 ─────▶│◀───── Block0 ─────▶│◀───── Block0 ─────▶│
        │      CopyIn        │      Compute       │      CopyOut       │
        │                    │                    │                    │
MTE2:   [════════════════════]                                        [═════...
Vector:                      [════════════════════]                      [═══...
MTE3:                                               [════════════════════]    ...

                                    ▲
                                    │
                          第2次 CopyIn 必须等这里完成后才能开始
                          (UB 空间需要先释放)

严格依赖关系：CopyIn(N) → Compute(N) → CopyOut(N) → CopyIn(N+1)

假设 CopyIn、Compute、CopyOut 各耗时 t:
- 每个数据块完整周期 = 3t
- Vector 时间利用率 = t / (3t) = 33%
- 等待时间占比 = 67%
```

### 2.2 解决方案：Unified Buffer DoubleBuffer

**原理：** 将待处理数据一分为二 (Tensor1, Tensor2)，交替使用

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  Unified Buffer                                                              │
│  ┌─────────────────────┐    ┌─────────────────────┐                        │
│  │      Block0         │    │      Block1         │                        │
│  │    (数据分块1)       │    │    (数据分块2)       │                        │
│  └─────────────────────┘    └─────────────────────┘                        │
└─────────────────────────────────────────────────────────────────────────────┘

时间轴 ──────────────────────────────────────────────────────────────────▶

MTE2:  [In0] [In1]  [In0] [In1]  ...
Vector:     [C0]   [C1]   [C0]  [C1]  ...
MTE3:           [Out0] [Out1] [Out0] ...

并行关系：
- Vector 计算 Block0 时，MTE2 搬运 Block1 入 UB
- Vector 计算 Block1 时，MTE3 搬运 Block0 出 UB

效果：
- Vector 时间利用率接近 100%
- 数据搬运时间被"隐藏"在计算时间中
```

### 2.3 适用条件

| 条件 | 适合使用 | 不适合使用 |
|------|----------|-----------|
| 数据量 | 大数据，需多次迭代 | 小数据，一次性处理完 |
| 搬运/计算比 | 搬运时间占比较高 | 计算时间远超搬运 |
| Vector 利用率 | 串行时利用率低 | 已接近饱和 |

## 3. Cube 计算流水线优化

### 3.1 Cube 数据流回顾

**基本计算：**

```
左矩阵:  GM ──(MTE2)──▶ L1 ──┐
                            │
右矩阵:  GM ──(MTE2)──▶ L1 ──(MTE1)──▶ L0B ──┐
                                           │
Fixpipe Buffer:                             ├──▶ Cube ──▶ L1
    GM ──(MTE2)──▶ L1 ──(FixPipe)──▶ FP ────┘
```

**累加计算：**

```
累加数据 (中间结果，已在 L1):
    L1 ────────────────────────────────────┤
                                           │
左矩阵 + 右矩阵 + FP + 累加数据 ─────────────┼──▶ Cube ──▶ L1 (累加结果)
```

### 3.2 问题：Cube 串行执行导致空闲

```
数据流: GM ──(MTE2)──▶ L1 ──(MTE1)──▶ L0B ──(Cube)──▶ L1

时间轴 ──────────────────────────────────────────────────────────────────────────────▶

        │◀───────────── Block A ─────────────────────▶│◀───────────── Block B ──────...
        │  Load右A  │  Load左A  │      L1→L0B     │    MMAD    │  Load右B  │  Load左A  │...
        │           │           │    (与Load左并行) │            │           │           │
MTE2:   [═══════════][══════════]                                         [═══════════]...
MTE1:                [════════════════]                                            [...
Cube:                                 [══════════════]                               ...

                                             ▲
                                             │
                                  MMAD 必须等两个条件都满足：
                                  1. Load左 完成 → 左矩阵在 L1
                                  2. L1→L0B 完成 → 右矩阵在 L0B

                                  Block B 的 Load右 必须等 Block A 的
                                  MMAD 完成 (L1 空间需要释放)

依赖关系：

  Load右(N) ──完成──▶ L1→L0B(N) ──┬──完成──▶ MMAD(N) ──完成──▶ Load右(N+1)
              │                   │
              └──▶ Load左(N) ─────┘
                    (MTE2 与 MTE1 并行)

若 t(右) = t(左) = t(L1→L0B) = t(MMAD) = t:
- 每个数据块完整周期 = t + max(t, t) + t = 3t
- Cube 时间利用率 = t / (3t) = 33%
```

### 3.3 解决方案：L1 + L0B 多级 DoubleBuffer

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  L1 Buffer (左矩阵/右矩阵/累加)                                               │
│  ┌─────────────────────┐    ┌─────────────────────┐                        │
│  │   Block0 (左矩阵)    │    │   Block1 (左矩阵)    │ ◀── Ping-Pong        │
│  └─────────────────────┘    └─────────────────────┘                        │
│  ┌─────────────────────┐    ┌─────────────────────┐                        │
│  │   Block0 (右矩阵)    │    │   Block1 (右矩阵)    │ ◀── Ping-Pong        │
│  └─────────────────────┘    └─────────────────────┘                        │
│  ┌─────────────────────┐    ┌─────────────────────┐                        │
│  │   Block0 (累加结果)  │    │   Block1 (累加结果)  │ ◀── Ping-Pong        │
│  └─────────────────────┘    └─────────────────────┘                        │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│  L0B Buffer (右矩阵)                                                         │
│  ┌─────────────────────┐    ┌─────────────────────┐                        │
│  │   Block0 (右矩阵)    │    │   Block1 (右矩阵)    │ ◀── Ping-Pong        │
│  └─────────────────────┘    └─────────────────────┘                        │
└─────────────────────────────────────────────────────────────────────────────┘

时间轴 ──────────────────────────────────────────────────────────────────▶

MTE2 (GM→L1):   [右0] [左0] [右1] [左1] [右0] [左0] ...
MTE1 (L1→L0B):      [右0→L0B] [右1→L0B] [右0→L0B] [右1→L0B] ...
Cube (MMAD):            [C0]  [C1]  [C0]  [C1] ...

三级流水线并行：
┌───────────────────────────────────────────────────────────────────────────┐
│  当 Cube 计算 Block[i] 时：                                                │
│    - MTE1 正在搬运 Block[i+1] 的右矩阵 L1→L0B                              │
│    - MTE2 正在搬运 Block[i+2] 的数据 GM→L1                                 │
│                                                                           │
│  三级并行: MTE2 ∥ MTE1 ∥ Cube                                              │
└───────────────────────────────────────────────────────────────────────────┘

效果：
- Cube 时间利用率接近 100%
- 数据搬运延迟被完全隐藏
```

### 3.4 累加优化：L1 中间结果复用

**场景：** 分块矩阵乘法 C = A × B，K 维度分块累加

**优化前 (每次结果写回 GM):**

```
Cube 计算 A1×B1 → L1 → MTE3 → GM
Cube 计算 A2×B2 → L1 → MTE3 → GM
Vector 从 GM 读取累加 → UB → Add → GM

问题：多次 GM 读写，带宽浪费
```

**优化后 (L1 中间结果复用):**

```
Cube 计算 A1×B1 → L1 (中间结果1)
Cube 计算 A2×B2 + L1(中间结果1) → L1 (中间结果2)  ◀── 直接在 L1 累加
Cube 计算 A3×B3 + L1(中间结果2) → L1 (最终结果)
最终结果 L1 → MTE3 → GM (只需一次写回)

优势：消除中间结果的 GM 读写，大幅减少带宽压力
```

**数据流图：**

```
左矩阵 (新):  GM ──(MTE2)──▶ L1 ──┐
                                │
右矩阵 (新):  GM ──(MTE2)──▶ L1 ──(MTE1)──▶ L0B ──┐
                                                │
累加数据 (上次结果，已在 L1):                    │
    L1 ─────────────────────────────────────────┤
                                                │
                                                ├──▶ Cube ──▶ L1
                                                │    (累加结果)
                                                │
最终结果: L1 ──(MTE3)──▶ GM ◀───────────────────┘
```

### 3.5 矩阵驻留策略

**场景：** A × B，其中 A 较大，B 较小

**假设：** L1 = 512KB，A = 992KB，B = 16KB

**策略：** 让较小的 B 矩阵完全驻留在 L1，只分块搬运 A 矩阵

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  L1 Buffer 分配：                                                            │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  B 矩阵 (16KB) - 完全驻留，不分块                                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────┐    ┌─────────────────────┐                        │
│  │  A 分块0 (248KB)     │    │  A 分块1 (248KB)     │ ◀── DoubleBuffer      │
│  └─────────────────────┘    └─────────────────────┘                        │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  累加结果区                                                           │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────────────┘

执行流程：
1. 一次性加载 B 矩阵到 L1 (16KB)
2. 分块加载 A 矩阵 (每次 248KB，DoubleBuffer)
3. 每块 A 与驻留的 B 进行矩阵乘
4. 结果在 L1 中累加

优势：
- B 矩阵只搬运一次，大幅减少 GM→L1 带宽压力
- 适用于 A ≫ B 或 B ≫ A 的不对称矩阵乘法
```

## 4. Vector + Cube 级联优化

### 4.1 级联场景概览

在实际神经网络算子中，Vector 计算和 Cube 计算经常需要配合使用。通过级联优化，可以避免中间数据经过 GM 中转，大幅减少带宽压力。

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                         三种级联模式                                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   模式1: Vector → Cube (独立算子预处理 + 矩阵乘)                               │
│   ────────────────────────────────────────                                  │
│   场景: LayerNorm、转置等独立算子预处理后进行矩阵乘                              │
│   关键通路: MTE3 (UB → L1)                                                   │
│                                                                             │
│   模式2: Cube → Vector (矩阵乘 + 独立算子后处理)                               │
│   ────────────────────────────────────────                                  │
│   场景: 矩阵乘后进行 Softmax、LayerNorm 等独立算子后处理                         │
│   关键通路: FixPipe (L1 → UB)                                                │
│                                                                             │
│   模式3: Vector → Cube → Vector (双向级联)                                    │
│   ──────────────────────────────────────────                                │
│   场景: 完整算子流水线，独立算子预处理+矩阵乘+独立算子后处理                        │
│   关键通路: MTE3 + FixPipe                                                   │
│                                                                             │
│   ⚠️ 注意: 量化、ReLU 等可通过 FixPipe 随路处理，不属于级联                       │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**级联模式对比：**

| 模式 | 数据流向 | 关键通路 | 典型场景 |
|------|----------|----------|----------|
| Vector → Cube | UB → L1 → L0B → Cube | MTE3: UB→L1 | LayerNorm→矩阵乘、转置→矩阵乘 |
| Cube → Vector | Cube → L1 → UB | FixPipe: L1→UB | 矩阵乘→Softmax、矩阵乘→一般计算 |
| 双向级联 | UB ↔ L1 | MTE3 + FixPipe | LayerNorm→矩阵乘→激活、Attention |

> ⚠️ **重要区分**：量化、激活等操作可通过 FixPipe 随路完成，无需独立 Vector 阶段。详见 [硬件架构规格 - FixPipe 随路处理能力](../npu-hardware-architecture/npu_arch_spec.md#53-fixpipe-随路处理能力)。

---

### 4.2 Vector → Cube 级联 (预处理 + 矩阵乘)

**场景：** 矩阵数据来自 Vector 计算（如量化、转置、重排等预处理）

**优化前 (经过 GM 中转):**

```
Vector 计算 → UB → MTE3 → GM → MTE2 → L1 → Cube

问题：Vector 结果写回 GM，Cube 再从 GM 读取，带宽浪费
```

**优化后 (直接级联):**

```
Vector 计算 → UB → MTE3 → L1 → MTE1 → L0B → Cube

优势：完全跳过 GM，通过内部存储直接传递数据
```

**数据流图：**

```
左矩阵 (来自 Vector):
    UB ──(MTE3)──▶ L1 ──┐
                        │
右矩阵 (来自 Vector):    │
    UB ──(MTE3)──▶ L1 ──(MTE1)──▶ L0B ──┐
                                       │
Fixpipe Buffer:                         ├──▶ Cube ──▶ L1
    GM ──(MTE2)──▶ L1 ──(FixPipe)──▶ FP ─┘

(UB 数据来自 Vector 计算，无需经过 GM)
```

**流水线时间轴：**

```
时间轴 ──────────────────────────────────────────────────────────────────────▶

        │◀───── Block0 ─────▶│◀───── Block1 ─────▶│
        │  Vector(预处理)    │   Cube(矩阵乘)     │

MTE2:   [════════════════════]                                  ...  (加载FP参数)
Vector:      [════════════════════]                             ...  (预处理Block0)
MTE3:             [════════════════════]                        ...  (UB→L1)
MTE1:                  [════════════════════]                   ...  (L1→L0B)
Cube:                       [════════════════════]              ...  (MMAD Block0)
Vector:                                        [════════════════════]  (预处理Block1)

两级流水线并行：
┌───────────────────────────────────────────────────────────────────────────┐
│  当 Cube 计算 Block[i] 时：                                                │
│    - Vector 正在预处理 Block[i+1]                                          │
│                                                                           │
│  两级并行: Vector ∥ Cube                                                   │
└───────────────────────────────────────────────────────────────────────────┘
```

**DoubleBuffer 应用：**

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  Vector → Cube 级联时的 DoubleBuffer 策略                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   Unified Buffer:                                                           │
│   ┌─────────────────────┐    ┌─────────────────────┐                       │
│   │  Block0 (预处理结果) │    │  Block1 (预处理结果) │  ◀── Ping-Pong       │
│   └─────────────────────┘    └─────────────────────┘                       │
│                                                                             │
│   L1 Buffer:                                                                │
│   ┌─────────────────────┐    ┌─────────────────────┐                       │
│   │  Block0 (左/右矩阵)  │    │  Block1 (左/右矩阵)  │  ◀── Ping-Pong       │
│   └─────────────────────┘    └─────────────────────┘                       │
│                                                                             │
│   同步点:                                                                    │
│   - Vector 完成 Block[i] 预处理 ──SetFlag──▶ MTE3 开始搬运                  │
│   - MTE3 完成搬运 ──SetFlag──▶ Cube 开始计算                                │
│   - Cube 完成计算 ──SetFlag──▶ Vector 可以覆盖该 UB Buffer                  │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 4.3 Cube → Vector 级联 (矩阵乘 + 独立 Vector 后处理)

**场景：** 矩阵乘结果需要进行**独立 Vector 算子**后处理（如 Softmax、LayerNorm、一般向量计算等）

> ⚠️ **注意**：量化、ReLU 激活等操作可通过 FixPipe 随路完成（见 [硬件架构规格 - FixPipe 随路处理能力](../npu-hardware-architecture/npu_arch_spec.md#53-fixpipe-随路处理能力)），无需此级联模式

**优化前 (经过 GM 中转):**

```
Cube 计算 → L1 → MTE3 → GM → MTE2 → UB → Vector(后处理) → UB → MTE3 → GM

问题：中间结果写回 GM 再读取，带宽浪费
```

**优化后 (直接级联):**

```
Cube 计算 → L1 → FixPipe → UB → Vector(后处理) → UB → MTE3 → GM

优势：Cube 结果通过 FixPipe 直通 UB，完全跳过 GM
```

**数据流图：**

```
左矩阵:  GM ──(MTE2)──▶ L1 ──┐
                            │
右矩阵:  GM ──(MTE2)──▶ L1 ──(MTE1)──▶ L0B ──┐
                                           │
Fixpipe Buffer:                             ├──▶ Cube ──▶ L1 ──(FixPipe)──▶ UB
    GM ──(MTE2)──▶ L1 ──(FixPipe)──▶ FP ────┘              │
                                                           │
Vector 后处理:                                             ▼
                                                UB ──(Vector)──▶ UB ──(MTE3)──▶ GM

关键通路: L1 → (FixPipe) → UB
```

**流水线时间轴：**

```
时间轴 ──────────────────────────────────────────────────────────────────────▶

        │◀───── Block0 ─────▶│◀───── Block1 ─────▶│
        │   Cube(矩阵乘)     │  Vector(独立算子)  │

MTE2:   [════════════════════]                                  ...
MTE1:        [════════════════════]                              ...
Cube:             [════════════════════]                         ...  (MMAD Block0)
FixPipe:               [════════════════════]                    ...  (L1→UB)
Vector:                     [════════════════════]               ...  (独立算子Block0，如Softmax)
MTE3:                           [════════════════════]           ...  (UB→GM)
Cube:                                              [════════════════════]  (MMAD Block1)

两级流水线并行：
┌───────────────────────────────────────────────────────────────────────────┐
│  当 Vector 执行独立算子 Block[i] 时：                                       │
│    - Cube 正在计算 Block[i+1]                                              │
│                                                                           │
│  两级并行: Cube ∥ Vector                                                   │
└───────────────────────────────────────────────────────────────────────────┘
```

### 4.4 双向级联 (Vector → Cube → Vector)

**场景：** 完整的算子流水线，包括前置独立 Vector 算子、矩阵乘和后置独立 Vector 算子

**典型应用：**
- LayerNorm → MatMul → Softmax (Transformer Attention)
- Add → MatMul → GELU (ResNet/Transformer FFN)
- Attention 中的 QK^T → Softmax → AV

> ⚠️ **注意**：量化推理 (Quantize → MatMul → Dequantize) 中的量化和反量化可通过 FixPipe 随路完成（见 [硬件架构规格 - FixPipe 随路处理能力](../npu-hardware-architecture/npu_arch_spec.md#53-fixpipe-随路处理能力)），**不需要**双向级联

**数据流图：**

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    Vector → Cube → Vector 完整级联                           │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   阶段1: Vector 预处理                                                       │
│   ────────────────────                                                      │
│   GM ──(MTE2)──▶ UB ──(Vector)──▶ UB ──(MTE3)──▶ L1                        │
│                                                   │                         │
│   阶段2: Cube 矩阵乘                          ▼                         │
│   ────────────────────                                                      │
│   L1 ──(MTE1)──▶ L0B ──┐                                                    │
│                        │                                                    │
│   L1 ──────────────────┼──▶ Cube ──▶ L1                                    │
│                        │                                                    │
│   阶段3: Vector 后处理  │                                                    │
│   ────────────────────│                                                    │
│   L1 ──(FixPipe)──▶ UB ──(Vector)──▶ UB ──(MTE3)──▶ GM                     │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**三级流水线时间轴：**

```
时间轴 ──────────────────────────────────────────────────────────────────────────────▶

        │◀───── Stage1 ─────▶│◀───── Stage2 ─────▶│◀───── Stage3 ─────▶│
        │  Vector(预处理)    │   Cube(矩阵乘)     │  Vector(后处理)    │

MTE2:   [════════════════════]                                            ...
Vector:      [════════════════════]                                       ...  (预处理B0)
MTE3:            [════════════════════]                                   ...  (UB→L1)
MTE1:                 [════════════════════]                              ...  (L1→L0B)
Cube:                      [════════════════════]                         ...  (MMAD B0)
FixPipe:                        [════════════════════]                    ...  (L1→UB)
Vector:                              [════════════════════]               ...  (后处理B0)
MTE3:                                    [════════════════════]           ...  (UB→GM)

三级流水线并行：
┌───────────────────────────────────────────────────────────────────────────┐
│  当 Vector(后处理) Block[i] 时：                                           │

│    - Vector(预处理) 正在处理 Block[i+2]                                    │

### 4.6 典型算子示例

#### 示例1: LayerNorm → MatMul (Vector→Cube 模式)

```
场景: Transformer 前馈层，LayerNorm 后进行线性变换

数据流:
  GM ──(MTE2)──▶ UB ──(Vector:LayerNorm)──▶ UB ──(MTE3)──▶ L1
                                                          │
                                                    ┌─────┘
                                                    │
  GM ──(MTE2)──▶ L1 ──(MTE1)──▶ L0B ────────────────┼──▶ Cube ──▶ L1 ──▶ GM
                                        (权重矩阵)   │

关键通路: MTE3 (UB → L1)

优化效果:
- LayerNorm 结果直接传递给 Cube，无需经过 GM
- 适用于 Transformer 的 FFN 层
```

#### 示例2: MatMul → Softmax (Cube→Vector 模式)

```
场景: 矩阵乘后进行 Softmax 计算（独立 Vector 算子）

数据流:
  GM ──(MTE2)──▶ L1 ──┐
                      │
  GM ──(MTE2)──▶ L1 ──(MTE1)──▶ L0B ──┐
                                     │
  GM ──(MTE2)──▶ L1 ──(FixPipe)──▶ FP ┼──▶ Cube ──▶ L1 ──(FixPipe)──▶ UB
                                                       │
                                                       ▼
                                              UB ──(Vector:Softmax)──▶ UB ──▶ GM

关键通路: FixPipe (L1 → UB)

优化效果:
- Cube 结果通过 FixPipe 直通 UB
- Vector 在 UB 上进行 Softmax 计算（独立算子，无法随路处理）
- 完全跳过 GM 中转

说明: Softmax 是复杂计算，无法通过 FixPipe 随路处理，需要独立 Vector 阶段
```

#### 示例3: MatMul + FixPipe 随路处理 (无级联)

```
场景: 量化推理，INT8 矩阵乘的量化和反量化通过 FixPipe 随路完成

数据流:
  输入数据 (FP32):
  GM ──(MTE2)──▶ L1

  矩阵乘 + 随路处理:
  L1 ──┐
       │
  L1 ──(MTE1)──▶ L0B ──┐
                       │
  Fixpipe Buffer ──────┼──▶ Cube ──▶ L1 ──(FixPipe:反量化)──▶ GM
  (量化参数)            │      │
                       │      └── 随路完成: INT8计算 + 反量化
                       │
  关键: 量化和反量化通过配置 Fixpipe Buffer 参数完成，无需独立 Vector 阶段

优化效果:
- 量化和反量化由 FixPipe 随路完成，视为 Cube 计算单元能力
- 无需 Vector → Cube → Vector 三级级联
- 数据流简洁: GM → L1 → Cube (+FixPipe) → GM

对比错误理解:
  ✗ Quantize → MatMul → Dequantize 需要双向级联
  ✓ 量化和反量化由 FixPipe 随路处理，只有纯 Cube 计算
```

#### 示例4: QK^T → Softmax → AV (Attention, 多阶段双向级联)

```
场景: Self-Attention 核心计算

阶段1: QK^T (Cube 矩阵乘)
  Q ──▶ L1 ──┐
             │
  K ──▶ L1 ──(MTE1)──▶ L0B ──┼──▶ Cube ──▶ L1 (QK^T 结果)

阶段2: Softmax (Vector 后处理 + Cube 矩阵乘准备)
  L1(QK^T) ──(FixPipe)──▶ UB ──(Vector:Softmax)──▶ UB ──(MTE3)──▶ L1

阶段3: AV (Cube 矩阵乘)
  Softmax结果 ──▶ L1 ──┐
                       │
  V ──▶ L1 ──(MTE1)──▶ L0B ──┼──▶ Cube ──▶ L1 ──▶ GM

完整数据流:
  ┌─────────────────────────────────────────────────────────────────────┐
  │                                                                     │
  │   Q,K ──▶ Cube(QK^T) ──▶ Vector(Softmax) ──▶ Cube(AV) ──▶ GM       │
  │            │                │                │                     │
  │            │                │                │                     │
  │         L1→UB            UB→L1            L1→GM                   │
  │        (FixPipe)        (MTE3)          (MTE3)                    │
  │                                                                     │
  └─────────────────────────────────────────────────────────────────────┘

优化效果:
- QK^T 结果通过 FixPipe 直通 UB 进行 Softmax
- Softmax 结果通过 MTE3 直通 L1 进行 AV 计算
- 只有最终结果写回 GM
```

**算子示例总结：**

| 算子组合 | 级联模式 | 关键通路 | 应用场景 |
|----------|----------|----------|----------|
| LayerNorm → MatMul | Vector→Cube | MTE3: UB→L1 | Transformer FFN |
| MatMul → Softmax | Cube→Vector | FixPipe: L1→UB | Attention |
| MatMul + FixPipe | 无级联 | FixPipe 随路 | 量化推理、简单激活 |
| QK^T → Softmax → AV | 多阶段双向 | FixPipe + MTE3 | Self-Attention |

> ⚠️ **重要**：量化推理 (INT8 MatMul) 中的量化和反量化由 FixPipe 随路处理，不需要双向级联

## 5. 同步机制与流水线控制

### 5.1 同步的必要性

不同指令序列可以并行执行，但可能存在数据依赖：

```
示例：Cube 计算 Block0 需要等待 MTE1 搬运完成

MTE1 序列: [搬运 Block0 到 L0B] ────────────────────────┐
                                                      │
Cube 序列:                    [使用 L0B 数据计算] ◀────┘ 必须等待！

如果没有同步，Cube 可能在数据未就绪时开始计算 → 错误结果
```

### 5.2 同步机制

**同一指令序列内：PipeBarrier**

```
Op1 ──▶ Op2 ──▶ [PipeBarrier] ──▶ Op3
                     │
                     ▼
           等待 Op1, Op2 全部完成后
           才开始执行 Op3
```

**不同指令序列间：SetFlag / WaitFlag**

```
MTE1 序列:                         Cube 序列:
[搬运数据] ──▶ SetFlag ────────▶   WaitFlag ──▶ [计算]
                   │                    │
                   ▼                    ▼
           搬运完成,            等待 Flag=1,
           设置 Flag=1         清零 Flag, 继续执行

⚠️ SetFlag 和 WaitFlag 必须成对使用，参数完全一致
```

### 5.3 DoubleBuffer 同步示例

```
时间轴 ──────────────────────────────────────────────────────────────────▶

MTE2:  [Load0] ──SetFlag(0)── [Load1] ──SetFlag(1)── [Load0] ...
MTE1:            WaitFlag(0) [MTE1_0] SetFlag(A0) WaitFlag(1) [MTE1_1]...
Cube:                         WaitFlag(A0) [Cube0] SetFlag(C0) ...

同步链：
MTE2 完成 Load0 ──▶ MTE1 开始搬运到 L0B
MTE1 完成搬运 ──▶ Cube 开始计算
Cube 完成计算 ──▶ MTE2 可以覆盖该 buffer (下一轮)

关键点：
- 每个 Block 有独立的 EventID
- SetFlag/WaitFlag 成对使用
- 使用后释放 EventID
```

## 7. 关键参数与约束

### 7.1 DoubleBuffer 容量计算

**Vector DoubleBuffer (UB):**

```
Block0_Size + Block1_Size ≤ UB_Size
Block_Size = Tile_M × Tile_N × sizeof(dtype)

推荐：Block0 = Block1 = UB_Size / 2
```

**Cube DoubleBuffer (L1 + L0B):**

```
L1 分配：
  左矩阵 = stepM × stepKa × 2(DB) × sizeof(dtype)
  右矩阵 = stepN × stepKb × 2(DB) × sizeof(dtype)
  累加结果 = stepM × stepN × sizeof(dtype)
  总计 ≤ L1_Size

L0B 分配：
  右矩阵 = baseN × baseK × 2(DB) × sizeof(dtype) ≤ L0B_Size

对齐约束：
  - L1: 32 Byte 对齐
  - L0B: 512 Byte 对齐
  - 分形大小对齐 (16×16 块)
```

### 7.2 存储单元对齐要求

| 存储单元 | 对齐要求 | 说明 |
|----------|----------|------|
| Unified Buffer | 32 Byte (DataBlock) | 向量计算数据块 |
| L1 Buffer | 32 Byte | 推荐 FRACTAL_NZ 格式 |
| L0B Buffer | **512 Byte** (16×16×float16) | 格式转换 NZ→ZN |
| BiasTable Buffer | 64 Byte | 矩阵计算 Bias |
| Fixpipe Buffer | 64 Byte | 量化/ReLU 参数 |

**注意事项：**

- 若 L1 剩余空间 < 1 分形 → 硬件异常
- L0B 推荐格式: FRACTAL_ZN (针对右矩阵优化)
- Vector 计算首地址必须 32 Byte 对齐

│   │  - 尽量减少 if/else 分支判断，保持代码路径简洁                         │   │
│   │  - 保证代码段被缓存在 ICache (代码 < 16K/32K)                         │   │
│   │  - ICache 以 2K 为单位加载，避免频繁跳转                              │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   DCache 优化 (数据缓存):                                                    │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  - 尽量减少变量运算                                                   │   │
│   │  - 保证数据段被缓存在 DCache                                          │   │
│   │  - DCache 数据首地址与 Cache Line (64B) 对齐                          │   │
│   │  - DCache 以 64B 为单位加载，优化数据局部性                            │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 9.3 缓存与性能

```
Scalar 性能影响链:

代码分支多 → ICache 频繁加载 → Scalar 等待 → 指令发射延迟 ↑
                                     │
                                     ▼
                            Vector/Cube/MTE 等待指令

数据未对齐 → DCache 加载效率低 → Scalar 数据访问延迟 ↑
```

## 10. 总结

### 10.1 核心原理

利用不同指令队列的独立性和可并行性，将数据搬运与计算重叠执行，隐藏数据搬运延迟，最大化计算单元利用率。

### 10.2 优化技术汇总

| 优化技术 | 适用场景 | 关键存储单元 | 数据通路 |
|----------|----------|--------------|----------|
| UB DoubleBuffer | Vector 计算 | Unified Buffer | GM↔UB |
| L1 + L0B DoubleBuffer | Cube 矩阵乘 | L1, L0B | GM→L1→L0B |
| L1 累加结果复用 | 分块矩阵乘累加 | L1 | L1 内部 |
| 小矩阵长驻 L1 | 不对称矩阵乘 | L1 | 一次加载 |
| Vector→Cube 级联 | 独立算子预处理+矩阵乘 | UB → L1 | **MTE3: UB→L1** |
| Cube→Vector 级联 | 矩阵乘+独立算子后处理 | L1 → UB | **FixPipe: L1→UB** |
| FixPipe 随路处理 | 量化、简单激活 | Fixpipe Buffer | Cube 内部 |
| ICache/DCache 优化 | Scalar 性能 | Cache | 对齐+局部性 |

> ⚠️ **注意**: FixPipe 随路处理（量化、ReLU 等）视为 Cube 计算单元能力，不属于级联优化

### 10.3 性能收益

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| Vector 利用率 | ~33% | 接近 100% |
| Cube 利用率 | ~33% | 接近 100% |
| GM 读写次数 | 多 | 显著减少 |
| 级联场景 GM 访问 | 必须经过 | 完全跳过 |

### 10.4 **级联优化需正确使用数据通路**
   - Vector→Cube: 使用 MTE3 (UB → L1) → Cube 计算结果通过 FixPipe 直通 UB 进行后处理
   - Cube→Vector 级联优化需要将 Cube 结果通过 Mte3(UB → L1) 后再搬运到 cube 进行矩阵乘
   - 量化推理不需要 Cube→vector级联，    - 矩阵乘后需要 Softmax/L归一化等复杂后处理
   - MatMul → Softmax等简单激活 →Cube→Vector级联
   - Attention: 只有独立 Vector算阶段是双向级联才需要真正的级联优化

### 10.5 **DoubleBuffer 不总是带来收益**
   - 数据量小、一次性处理完时可能适得其反
   - 计算时间远超搬运时间时收益有限

### 10.6. **合理设置同步点**
   - 避免数据竞争
   - SetFlag/WaitFlag 必须成对使用

### 10.7 **容量计算需考虑对齐要求**
   - L1: 32 Byte
   - L0B: **512 Byte**
   - UB: 32 Byte
   - BiasTable/Fixpipe Buffer: **64 Byte**
   - 分形大小 (16×16)

### 10.8 **级联优化需正确使用数据通路**
   - Vector→Cube: 使用 MTE3 (UB→L1)
   - Cube→Vector: 使用 FixPipe (L1→UB)，仅用于独立 Vector 算子
   - 量化、ReLU 等可通过 FixPipe 随路处理，无需级联
   - 避免不必要的 GM 中转

### 10.9 **Bank 冲突可能抵消 DoubleBuffer 收益**
   - 合理分配 Block 位置
   - 关注并发访问模式

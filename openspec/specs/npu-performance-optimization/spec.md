## 1. 性能优化的硬件基础

### 1.1 指令队列并行性 —— 优化的基石

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    指令队列并行性                                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │                    Scalar (指令分类与调度)                           │   │
│   └─┬───────┬───────┬───────┬───────┬───────┬───────┬───┘               │   │
│     │       │       │       │       │       │       │                    │   │
│     ▼       ▼       ▼       ▼       ▼       ▼       ▼                    │   │
│   ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐              │   │
│   │Scalar│ │Vector│ │Cube │ │MTE1 │ │MTE2 │ │MTE3 │ │FixPipe│              │   │
│   │ 序列 │ │ 序列 │ │ 序列 │ │ 序列 │ │ 序列 │ │ 序列 │ │ 序列 │              │   │
│   └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘ └──┬──┘              │   │
│      │       │       │       │       │       │       │                    │   │
│      └───────┴───────┴───────┴───────┴───────┴───────┘                    │   │
│                          │                                                  │   │
│                          ▼                                                  │   │
│              不同序列间可并行执行                                            │   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**关键洞察：**

- Scalar/Vector/Cube/MTE1/MTE2/MTE3/FixPipe 各为独立指令序列
- 任意两个不同序列之间都可以并行执行
- 这是 DoubleBuffer 优化的硬件理论基础

## 2. Vector 计算流水线优化

### 2.1 问题：串行执行导致 Vector 空闲

```
数据流: GM ──(MTE2)──▶ UB ──(Vector)──▶ UB ──(MTE3)──▶ GM

时间轴 ────────────────────────────────────────────────────────────────────────▶

        │◀───── Block0 ─────▶│◀───── Block0 ─────▶│◀───── Block0 ─────▶│
        │      CopyIn        │      Compute       │      CopyOut       │
        │                    │                    │                    │
MTE2:   [════════════════════]                                        [═════...
Vector:                      [════════════════════]                      [═══...
MTE3:                                               [════════════════════]    ...

                                    ▲
                                    │
                          第2次 CopyIn 必须等这里完成后才能开始
                          (UB 空间需要先释放)

严格依赖关系：CopyIn(N) → Compute(N) → CopyOut(N) → CopyIn(N+1)

假设 CopyIn、Compute、CopyOut 各耗时 t:
- 每个数据块完整周期 = 3t
- Vector 时间利用率 = t / (3t) = 33%
- 等待时间占比 = 67%
```

### 2.2 解决方案：Unified Buffer DoubleBuffer

**原理：** 将待处理数据一分为二 (Tensor1, Tensor2)，交替使用

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  Unified Buffer                                                              │
│  ┌─────────────────────┐    ┌─────────────────────┐                        │
│  │      Block0         │    │      Block1         │                        │
│  │    (数据分块1)       │    │    (数据分块2)       │                        │
│  └─────────────────────┘    └─────────────────────┘                        │
└─────────────────────────────────────────────────────────────────────────────┘

时间轴 ──────────────────────────────────────────────────────────────────▶

MTE2:  [In0] [In1]  [In0] [In1]  ...
Vector:     [C0]   [C1]   [C0]  [C1]  ...
MTE3:           [Out0] [Out1] [Out0] ...

并行关系：
- Vector 计算 Block0 时，MTE2 搬运 Block1 入 UB
- Vector 计算 Block1 时，MTE3 搬运 Block0 出 UB

效果：
- Vector 时间利用率接近 100%
- 数据搬运时间被"隐藏"在计算时间中
```

### 2.3 适用条件

| 条件 | 适合使用 | 不适合使用 |
|------|----------|-----------|
| 数据量 | 大数据，需多次迭代 | 小数据，一次性处理完 |
| 搬运/计算比 | 搬运时间占比较高 | 计算时间远超搬运 |
| Vector 利用率 | 串行时利用率低 | 已接近饱和 |

## 3. Cube 计算流水线优化

### 3.1 Cube 数据流回顾

**基本计算：**

```
左矩阵:  GM ──(MTE2)──▶ L1 ──┐
                            │
右矩阵:  GM ──(MTE2)──▶ L1 ──(MTE1)──▶ L0B ──┐
                                           │
Fixpipe Buffer:                             ├──▶ Cube ──▶ L1
    GM ──(MTE2)──▶ L1 ──(FixPipe)──▶ FP ────┘
```

**累加计算：**

```
累加数据 (中间结果，已在 L1):
    L1 ────────────────────────────────────┤
                                           │
左矩阵 + 右矩阵 + FP + 累加数据 ─────────────┼──▶ Cube ──▶ L1 (累加结果)
```

### 3.2 问题：Cube 串行执行导致空闲

```
数据流: GM ──(MTE2)──▶ L1 ──(MTE1)──▶ L0B ──(Cube)──▶ L1

时间轴 ──────────────────────────────────────────────────────────────────────────────▶

        │◀───────────── Block A ─────────────────────▶│◀───────────── Block B ──────...
        │  Load右A  │  Load左A  │      L1→L0B     │    MMAD    │  Load右B  │  Load左A  │...
        │           │           │    (与Load左并行) │            │           │           │
MTE2:   [═══════════][══════════]                                         [═══════════]...
MTE1:                [════════════════]                                            [...
Cube:                                 [══════════════]                               ...

                                             ▲
                                             │
                                  MMAD 必须等两个条件都满足：
                                  1. Load左 完成 → 左矩阵在 L1
                                  2. L1→L0B 完成 → 右矩阵在 L0B

                                  Block B 的 Load右 必须等 Block A 的
                                  MMAD 完成 (L1 空间需要释放)

依赖关系：

  Load右(N) ──完成──▶ L1→L0B(N) ──┬──完成──▶ MMAD(N) ──完成──▶ Load右(N+1)
              │                   │
              └──▶ Load左(N) ─────┘
                    (MTE2 与 MTE1 并行)

若 t(右) = t(左) = t(L1→L0B) = t(MMAD) = t:
- 每个数据块完整周期 = t + max(t, t) + t = 3t
- Cube 时间利用率 = t / (3t) = 33%
```

### 3.3 解决方案：L1 + L0B 多级 DoubleBuffer

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  L1 Buffer (左矩阵/累加)                                                     │
│  ┌─────────────────────┐    ┌─────────────────────┐                        │
│  │   Block0 (左矩阵)    │    │   Block1 (左矩阵)    │ ◀── Ping-Pong        │
│  └─────────────────────┘    └─────────────────────┘                        │
│  ┌─────────────────────┐    ┌─────────────────────┐                        │
│  │   Block0 (累加结果)  │    │   Block1 (累加结果)  │ ◀── Ping-Pong        │
│  └─────────────────────┘    └─────────────────────┘                        │
└─────────────────────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────────────────────┐
│  L0B Buffer (右矩阵)                                                         │
│  ┌─────────────────────┐    ┌─────────────────────┐                        │
│  │   Block0 (右矩阵)    │    │   Block1 (右矩阵)    │ ◀── Ping-Pong        │
│  └─────────────────────┘    └─────────────────────┘                        │
└─────────────────────────────────────────────────────────────────────────────┘

时间轴 ──────────────────────────────────────────────────────────────────▶

MTE2 (GM→L1):   [左0] [右0] [左1] [右1] [左0] [右0] ...
MTE1 (L1→L0B):      [0]   [1]   [0]   [1]   [0] ...
Cube (MMAD):            [C0]  [C1]  [C0]  [C1] ...

三级流水线并行：
┌───────────────────────────────────────────────────────────────────────────┐
│  当 Cube 计算 Block[i] 时：                                                │
│    - MTE1 正在搬运 Block[i+1] 的右矩阵 L1→L0B                              │
│    - MTE2 正在搬运 Block[i+2] 的数据 GM→L1                                 │
│                                                                           │
│  三级并行: MTE2 ∥ MTE1 ∥ Cube                                              │
└───────────────────────────────────────────────────────────────────────────┘

效果：
- Cube 时间利用率接近 100%
- 数据搬运延迟被完全隐藏
```

### 3.4 累加优化：L1 中间结果复用

**场景：** 分块矩阵乘法 C = A × B，K 维度分块累加

**优化前 (每次结果写回 GM):**

```
Cube 计算 A1×B1 → L1 → MTE3 → GM
Cube 计算 A2×B2 → L1 → MTE3 → GM
Vector 从 GM 读取累加 → UB → Add → GM

问题：多次 GM 读写，带宽浪费
```

**优化后 (L1 中间结果复用):**

```
Cube 计算 A1×B1 → L1 (中间结果1)
Cube 计算 A2×B2 + L1(中间结果1) → L1 (中间结果2)  ◀── 直接在 L1 累加
Cube 计算 A3×B3 + L1(中间结果2) → L1 (最终结果)
最终结果 L1 → MTE3 → GM (只需一次写回)

优势：消除中间结果的 GM 读写，大幅减少带宽压力
```

**数据流图：**

```
左矩阵 (新):  GM ──(MTE2)──▶ L1 ──┐
                                │
右矩阵 (新):  GM ──(MTE2)──▶ L1 ──(MTE1)──▶ L0B ──┐
                                                │
累加数据 (上次结果，已在 L1):                    │
    L1 ─────────────────────────────────────────┤
                                                │
                                                ├──▶ Cube ──▶ L1
                                                │    (累加结果)
                                                │
最终结果: L1 ──(MTE3)──▶ GM ◀───────────────────┘
```

### 3.5 矩阵驻留策略

**场景：** A × B，其中 A 较大，B 较小

**假设：** L1 = 512KB，A = 992KB，B = 16KB

**策略：** 让较小的 B 矩阵完全驻留在 L1，只分块搬运 A 矩阵

```
┌─────────────────────────────────────────────────────────────────────────────┐
│  L1 Buffer 分配：                                                            │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  B 矩阵 (16KB) - 完全驻留，不分块                                     │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│  ┌─────────────────────┐    ┌─────────────────────┐                        │
│  │  A 分块0 (248KB)     │    │  A 分块1 (248KB)     │ ◀── DoubleBuffer      │
│  └─────────────────────┘    └─────────────────────┘                        │
│                                                                             │
│  ┌─────────────────────────────────────────────────────────────────────┐   │
│  │  累加结果区                                                           │   │
│  └─────────────────────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────────────────────┘

执行流程：
1. 一次性加载 B 矩阵到 L1 (16KB)
2. 分块加载 A 矩阵 (每次 248KB，DoubleBuffer)
3. 每块 A 与驻留的 B 进行矩阵乘
4. 结果在 L1 中累加

优势：
- B 矩阵只搬运一次，大幅减少 GM→L1 带宽压力
- 适用于 A ≫ B 或 B ≫ A 的不对称矩阵乘法
```

## 4. Vector + Cube 级联优化

**场景：** 矩阵数据来自 Vector 计算（如量化、转置等预处理）

**优化前 (经过 GM 中转):**

```
Vector 计算 → UB → MTE3 → GM → MTE2 → L1 → Cube

问题：Vector 结果写回 GM，Cube 再从 GM 读取，带宽浪费
```

**优化后 (直接级联):**

```
Vector 计算 → UB → MTE3 → L1 → MTE1 → L0B → Cube

优势：完全跳过 GM，通过内部存储直接传递数据
```

**数据流图：**

```
左矩阵 (来自 Vector):
    UB ──(MTE3)──▶ L1 ──┐
                        │
右矩阵 (来自 Vector):    │
    UB ──(MTE3)──▶ L1 ──(MTE1)──▶ L0B ──┐
                                       │
Fixpipe Buffer:                         ├──▶ Cube ──▶ L1
    GM ──(MTE2)──▶ L1 ──(FixPipe)──▶ FP ─┘

(UB 数据来自 Vector 计算，无需经过 GM)
```

## 5. 同步机制与流水线控制

### 5.1 同步的必要性

不同指令序列可以并行执行，但可能存在数据依赖：

```
示例：Cube 计算 Block0 需要等待 MTE1 搬运完成

MTE1 序列: [搬运 Block0 到 L0B] ────────────────────────┐
                                                      │
Cube 序列:                    [使用 L0B 数据计算] ◀────┘ 必须等待！

如果没有同步，Cube 可能在数据未就绪时开始计算 → 错误结果
```

### 5.2 同步机制

**同一指令序列内：PipeBarrier**

```
Op1 ──▶ Op2 ──▶ [PipeBarrier] ──▶ Op3
                     │
                     ▼
           等待 Op1, Op2 全部完成后
           才开始执行 Op3
```

**不同指令序列间：SetFlag / WaitFlag**

```
MTE1 序列:                         Cube 序列:
[搬运数据] ──▶ SetFlag ────────▶   WaitFlag ──▶ [计算]
                   │                    │
                   ▼                    ▼
           搬运完成,            等待 Flag=1,
           设置 Flag=1         清零 Flag, 继续执行

⚠️ SetFlag 和 WaitFlag 必须成对使用，参数完全一致
```

### 5.3 DoubleBuffer 同步示例

```
时间轴 ──────────────────────────────────────────────────────────────────▶

MTE2:  [Load0] ──SetFlag(0)── [Load1] ──SetFlag(1)── [Load0] ...
MTE1:            WaitFlag(0) [MTE1_0] SetFlag(A0) WaitFlag(1) [MTE1_1]...
Cube:                         WaitFlag(A0) [Cube0] SetFlag(C0) ...

同步链：
MTE2 完成 Load0 ──▶ MTE1 开始搬运到 L0B
MTE1 完成搬运 ──▶ Cube 开始计算
Cube 完成计算 ──▶ MTE2 可以覆盖该 buffer (下一轮)

关键点：
- 每个 Block 有独立的 EventID
- SetFlag/WaitFlag 成对使用
- 使用后释放 EventID
```

## 7. 关键参数与约束

### 7.1 DoubleBuffer 容量计算

**Vector DoubleBuffer (UB):**

```
Block0_Size + Block1_Size ≤ UB_Size
Block_Size = Tile_M × Tile_N × sizeof(dtype)

推荐：Block0 = Block1 = UB_Size / 2
```

**Cube DoubleBuffer (L1 + L0B):**

```
L1 分配：
  左矩阵 = stepM × stepKa × 2(DB) × sizeof(dtype)
  右矩阵 = stepN × stepKb × 2(DB) × sizeof(dtype)
  累加结果 = stepM × stepN × sizeof(dtype)
  总计 ≤ L1_Size

L0B 分配：
  右矩阵 = baseN × baseK × 2(DB) × sizeof(dtype) ≤ L0B_Size

对齐约束：
  - L1: 32 Byte 对齐
  - L0B: 512 Byte 对齐
  - 分形大小对齐 (16×16 块)
```

### 7.2 存储单元对齐要求

| 存储单元 | 对齐要求 | 说明 |
|----------|----------|------|
| Unified Buffer | 32 Byte (DataBlock) | 向量计算数据块 |
| L1 Buffer | 32 Byte | 推荐 FRACTAL_NZ 格式 |
| L0B Buffer | **512 Byte** (16×16×float16) | 格式转换 NZ→ZN |
| BiasTable Buffer | 64 Byte | 矩阵计算 Bias |
| Fixpipe Buffer | 64 Byte | 量化/ReLU 参数 |

**注意事项：**

- 若 L1 剩余空间 < 1 分形 → 硬件异常
- L0B 推荐格式: FRACTAL_ZN (针对右矩阵优化)
- Vector 计算首地址必须 32 Byte 对齐

## 8. 搬运单元完整数据通路

### 8.1 MTE3 完整通路

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                       MTE3 数据通路                                          │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   Unified Buffer                                                            │
│        │                                                                    │
│        ├──▶ GM   (结果写回全局内存)                                          │
│        │                                                                    │
│        └──▶ L1   (Vector→Cube 级联，数据直通)  ◀── 关键优化通路              │
│                                                                             │
│   L1 Buffer                                                                 │
│        │                                                                    │
│        └──▶ GM   (Cube 结果写回)                                             │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**优化场景：**

- **UB → L1**：Vector 计算结果直接传递给 Cube，无需经过 GM
- 与 Vector+Cube 级联优化配合使用

### 8.2 FixPipe 完整通路

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                       FixPipe 数据通路                                       │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   L1 Buffer                                                                 │
│        │                                                                    │
│        ├──▶ L1         (内部格式/类型转换)                                   │
│        │                                                                    │
│        ├──▶ BT Buffer  (BiasTable，矩阵计算 Bias)                           │
│        │                                                                    │
│        ├──▶ FP Buffer  (量化参数、ReLU 参数)                                │
│        │                                                                    │
│        └──▶ UB        (Cube→Vector 直通)  ◀── 关键优化通路                  │
│                                                                             │
│   ⚠️ FixPipe 支持随路数据格式/类型转换                                        │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

**优化场景：**

- **L1 → UB**：Cube 计算结果直接传递给 Vector 进行后处理
- 对应硬件规格中的 Vector 数据流方式3：
  ```
  L1 数据通过 FixPipe 指令搬到 UB，Vector 计算从 UB 中取数据计算
  ```

### 8.3 完整级联数据流

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                    Vector ←→ Cube 完整级联数据流                             │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   Vector → Cube 级联:                                                       │
│                                                                             │
│   GM ──(MTE2)──▶ UB ──(Vector)──▶ UB ──(MTE3)──▶ L1 ──(MTE1)──▶ L0B        │
│                                                        │                    │
│                                                        ▼                    │
│                                                      Cube ──▶ L1            │
│                                                                             │
│                                                                             │
│   Cube → Vector 级联:                                                       │
│                                                                             │
│   ... ──▶ Cube ──▶ L1 ──(FixPipe)──▶ UB ──(Vector)──▶ UB ──(MTE3)──▶ GM    │
│                                                                             │
│                                                                             │
│   双向级联优势:                                                              │
│   - 完全跳过 GM 中转                                                         │
│   - 通过内部存储直接传递数据                                                  │
│   - 大幅减少带宽压力                                                         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 9. Unified Buffer Bank 冲突优化

### 9.1 Bank 冲突问题

当多个操作同时访问 Unified Buffer 的同一个 bank 或 bank group 时，会发生：

- **读写冲突**：读操作和写操作同时访问同一 bank
- **写写冲突**：多个写操作同时访问同一 bank
- **读读冲突**：多个读操作同时访问同一 bank

**后果：** 访问排队，降低性能

### 9.2 DoubleBuffer 场景下的 Bank 冲突

```
┌─────────────────────────────────────────────────────────────────────────────┐
│              DoubleBuffer 模式下的并发访问                                    │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   时间点 T:                                                                  │
│                                                                             │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │                    Unified Buffer                                    │   │
│   │  ┌─────────────────────┐    ┌─────────────────────┐                │   │
│   │  │      Block0         │    │      Block1         │                │   │
│   │  │   ◀── Vector 读取    │    │   ◀── MTE2 写入     │                │   │
│   │  │   ◀── MTE3 读取      │    │                     │                │   │
│   │  └─────────────────────┘    └─────────────────────┘                │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   并发操作:                                                                  │
│   - MTE2: 写入 Block1 (GM → UB)                                            │
│   - Vector: 读取 Block0 (计算)                                              │
│   - MTE3: 读取 Block0 (UB → GM)                                            │
│                                                                             │
│   若 Block0 和 Block1 映射到同一 bank group → 冲突!                         │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 9.3 优化策略

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      Bank 冲突优化策略                                        │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   1. 合理分配 Block 在 UB 中的位置                                           │
│      - 确保 Block0 和 Block1 位于不同 bank group                             │
│      - 使用编程模型提供的内存分配接口                                         │
│                                                                             │
│   2. 调整数据分块大小                                                        │
│      - 使 Block 大小为 bank 大小的整数倍                                     │
│      - 避免跨 bank 边界访问                                                  │
│                                                                             │
│   3. 优化访问时序                                                            │
│      - 通过同步机制控制访问顺序                                              │
│      - 减少同一时刻的并发访问数                                               │
│                                                                             │
│   ⚠️ 具体实现请参考硬件规格第9节和官方编程指南                                │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

## 10. Scalar 缓存优化

### 10.1 缓存规格

| 缓存 | 大小 | 加载单位 |
|------|------|----------|
| ICache | 16K / 32K | 2K |
| DCache | 16K | Cache Line (64B) |

### 10.2 优化建议

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                      Scalar 缓存优化                                         │
├─────────────────────────────────────────────────────────────────────────────┤
│                                                                             │
│   ICache 优化 (指令缓存):                                                    │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  - 尽量减少 if/else 分支判断，保持代码路径简洁                         │   │
│   │  - 保证代码段被缓存在 ICache (代码 < 16K/32K)                         │   │
│   │  - ICache 以 2K 为单位加载，避免频繁跳转                              │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
│   DCache 优化 (数据缓存):                                                    │
│   ┌─────────────────────────────────────────────────────────────────────┐   │
│   │  - 尽量减少变量运算                                                   │   │
│   │  - 保证数据段被缓存在 DCache                                          │   │
│   │  - DCache 数据首地址与 Cache Line (64B) 对齐                          │   │
│   │  - DCache 以 64B 为单位加载，优化数据局部性                            │   │
│   └─────────────────────────────────────────────────────────────────────┘   │
│                                                                             │
└─────────────────────────────────────────────────────────────────────────────┘
```

### 10.3 缓存与性能

```
Scalar 性能影响链:

代码分支多 → ICache 频繁加载 → Scalar 等待 → 指令发射延迟 ↑
                                     │
                                     ▼
                            Vector/Cube/MTE 等待指令

数据未对齐 → DCache 加载效率低 → Scalar 数据访问延迟 ↑
```

## 11. 总结

### 11.1 核心原理

利用不同指令队列的独立性和可并行性，将数据搬运与计算重叠执行，隐藏数据搬运延迟，最大化计算单元利用率。

### 11.2 优化技术汇总

| 优化技术 | 适用场景 | 关键存储单元 | 数据通路 |
|----------|----------|--------------|----------|
| UB DoubleBuffer | Vector 计算 | Unified Buffer | GM↔UB |
| L1 + L0B DoubleBuffer | Cube 矩阵乘 | L1, L0B | GM→L1→L0B |
| L1 累加结果复用 | 分块矩阵乘累加 | L1 | L1 内部 |
| 小矩阵长驻 L1 | 不对称矩阵乘 | L1 | 一次加载 |
| Vector→Cube 级联 | 预处理+矩阵乘 | UB → L1 | **MTE3: UB→L1** |
| Cube→Vector 级联 | 矩阵乘+后处理 | L1 → UB | **FixPipe: L1→UB** |
| Bank 冲突优化 | DoubleBuffer 场景 | UB | 合理分配 |
| ICache/DCache 优化 | Scalar 性能 | Cache | 对齐+局部性 |

### 11.3 性能收益

| 指标 | 优化前 | 优化后 |
|------|--------|--------|
| Vector 利用率 | ~33% | 接近 100% |
| Cube 利用率 | ~33% | 接近 100% |
| GM 读写次数 | 多 | 显著减少 |
| 级联场景 GM 访问 | 必须经过 | 完全跳过 |

### 11.4 注意事项

1. **DoubleBuffer 不总是带来收益**
   - 数据量小、一次性处理完时可能适得其反
   - 计算时间远超搬运时间时收益有限

2. **合理设置同步点**
   - 避免数据竞争
   - SetFlag/WaitFlag 必须成对使用

3. **容量计算需考虑对齐要求**
   - L1: 32 Byte
   - L0B: **512 Byte**
   - UB: 32 Byte
   - BiasTable/Fixpipe Buffer: **64 Byte**
   - 分形大小 (16×16)

4. **级联优化需正确使用数据通路**
   - Vector→Cube: 使用 MTE3 (UB→L1)
   - Cube→Vector: 使用 FixPipe (L1→UB)
   - 避免不必要的 GM 中转

5. **Bank 冲突可能抵消 DoubleBuffer 收益**
   - 合理分配 Block 位置
   - 关注并发访问模式
